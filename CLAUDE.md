# Personal Preferences

## Tooling
- Use **bun** over npm or pnpm
- Use **cargo clippy** over cargo check
- Use **biome** over eslint and prettier

## TypeScript conventions
- Use double quotes `"` for strings
- Always use semicolons
- Use 4-space indentation

## Workflow
Follow this process for every task:
1. **Discuss** — align on requirements and approach
2. **Context sync** — make sure Claude and I share the same understanding
3. **Read specs** — if `SPECS.md` exists, read it and compare the current code against it before planning; if the code or task doesn't align, stop and ask why before proceeding
4. **Plan** — create a detailed implementation plan
5. **Assign tasks** — break work into tracked tasks
6. **Execute** — implement the plan
7. **Code review** — review the completed work
8. **QA / Unit test** — quick quality check or unit tests
9. **Report** — deliver a comprehensive summary
